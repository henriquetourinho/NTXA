IoT no Pasto:
O desafio da conectividade rural.

Henrique Tourinho

CTO & Founder @ NXTA

Conectar um servidor em um data center climatizado é trivial. Conectar um animal de 600kg que se move por vales, colinas e áreas de sombra de sinal no interior da Bahia é engenharia de verdade.

A maioria das soluções de "Agro 4.0" falha no Brasil por um motivo simples: elas assumem que o campo tem internet. O DeméterOne assume que não tem.

O Problema da Topografia

O sinal 4G/5G é linear. Uma colina interrompe a comunicação. Se usássemos modems convencionais, perderíamos dados de telemetria crítica (parto, doença, roubo) em 40% do tempo.

Além disso, o custo energético de buscar rede celular drenaria a bateria dos sensores em dias. Precisávamos de uma arquitetura Offline-First.

Arquitetura Mesh & Store-and-Forward

Desenvolvemos um protocolo proprietário sobre LoRa (Long Range) que permite que os animais funcionem como "repetidores" uns dos outros. Se a vaca A não tem sinal, mas vê a vaca B (que vê a antena), o pacote de dados salta.

"Não enviamos JSON. No pasto, cada byte custa bateria. Criamos um protocolo binário que comprime 24h de dados de saúde em payloads minúsculos."

Por que Rust no Embarcado?

C++ é o padrão, mas Rust nos dá a segurança de memória necessária para dispositivos que não podem ser reiniciados manualmente. Se o firmware falhar no meio do pasto, o custo operacional é altíssimo.

Abaixo, um exemplo simplificado de como gerenciamos o buffer de dados quando o animal está totalmente isolado (Store & Forward):

firmware/telemetry.rs 
use heapless::Vec;

struct TelemetryPacket {
    timestamp: u32,
    gps_lat: f32,
    gps_long: f32,
    activity_index: u8, // 0-255 (Repouso -> Stress)
}

// Armazena dados na memória flash se não houver LoRa
fn handle_telemetry(sensor: &SensorData, radio: &mut LoRaRadio) {
    let packet = TelemetryPacket::new(sensor);

    match radio.send(&packet) {
        Ok(_) => {
            println!("Packet sent immediately.");
        },
        Err(Error::NoNetwork) => {
            // Escrita segura na Flash (No-STD)
            flash_storage.buffer(packet).expect("Flash full panic");
            println!("Offline mode: Data buffered.");
        }
    }
}

O Resultado

Conseguimos reduzir o consumo de bateria em 60% comparado a soluções de mercado e garantimos que, mesmo que o animal fique 3 dias sem sinal, assim que ele entrar no raio de cobertura (ou cruzar com outro animal conectado), o histórico completo é sincronizado com a nuvem do DeméterOne.

Isso não é apenas rastreamento. É inteligência de borda aplicada onde ela é mais difícil de implementar.
