Existe uma regra de ouro na administração de sistemas: Se você precisa digitar o mesmo comando três vezes, automatize.

Ferramentas modernas como Ansible, Terraform e Kubernetes são incríveis. Mas elas adicionam complexidade. Na NXTA, nós amamos a simplicidade brutal do Bash. Para provisionar um novo nó de processamento para o DeméterOne ou um servidor de borda, usamos scripts que rodam em qualquer Debian "vanilla".

A Filosofia da Idempotência

Um bom script de automação não pode apenas "rodar". Ele deve ser idempotente. Isso significa que você pode executá-lo 1, 10 ou 100 vezes, e o resultado final será sempre o mesmo, sem quebrar configurações existentes ou duplicar arquivos.

"Shell script não é apenas colar comandos em um arquivo. É lógica de estado, tratamento de erros e garantia de consistência."

O Script de "Bootstrap" NXTA

Abaixo está uma versão higienizada do script que usamos para preparar um servidor Debian 12 recém-instalado. Ele atualiza o sistema, instala ferramentas base, configura o firewall e blinda o SSH.

setup-server.sh 
#!/bin/bash
# NXTA Server Bootstrap - Debian 12
# Encerra o script se qualquer comando falhar
set -e

NEW_USER="nxta_admin"
SSH_PORT=22022

echo "[+] Atualizando Repositórios..."
apt-get update && apt-get upgrade -y

echo "[+] Instalando Ferramentas Base..."
apt-get install -y curl git htop ufw fail2ban vim net-tools

echo "[+] Configurando Firewall (UFW)..."
# Libera portas essenciais antes de ativar
ufw allow $SSH_PORT/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

echo "[+] Blindando SSH..."
# Altera porta e proíbe login de root
sed -i "s/#Port 22/Port $SSH_PORT/" /etc/ssh/sshd_config
sed -i "s/PermitRootLogin yes/PermitRootLogin no/" /etc/ssh/sshd_config
systemctl restart ssh

echo "[OK] Servidor provisionado com sucesso."

Por que Debian?

Estabilidade é chata, e nós amamos isso. Em produção, você não quer surpresas. O ciclo de release do Debian e sua gestão de pacotes (`apt`) são previsíveis.

Quando você roda um script desses em 50 servidores, você sabe que todos terminarão no mesmo estado. Essa confiança é a base sobre a qual construímos produtos complexos como o BrHTTP.
